DATABASE TABLES
================================================================================

TABLE: users
  Columns:
    • id (INT, Primary Key)
    • first_name (VARCHAR 100)
    • middle_name (VARCHAR 100) - Optional
    • last_name (VARCHAR 100)
    • email (VARCHAR 150) - UNIQUE
    • student_id (VARCHAR 50) - UNIQUE
    • department (VARCHAR 100)
    • password (VARCHAR 255) - Hashed
    • has_voted (TINYINT 0/1)
    • registration_date (TIMESTAMP)

TABLE: votes
  Columns:
    • vote_id (INT, Primary Key)
    • student_id (VARCHAR 50) - Foreign Key
    • president (VARCHAR 100)
    • vice_president (VARCHAR 100)
    • secretary (VARCHAR 100)
    • timestamp (TIMESTAMP)

SECURITY FEATURES
================================================================================

✓ Password Security:
  • Hashed with password_hash() using PASSWORD_DEFAULT algorithm
  • Verified with password_verify()
  • Minimum 6 characters
  • Cannot be stored in plain text

✓ SQL Injection Prevention:
  • All queries use prepared statements
  • Parameterized queries with bind_param()
  • No string concatenation in SQL

✓ Authentication:
  • Session-based user validation
  • Automatic redirect if not logged in
  • Session variables: user_id, student_id, first_name, has_voted

✓ One-Vote Prevention:
  • Database flag: has_voted (0 = not voted, 1 = voted)
  • Checked before showing vote page
  • Updated after vote submission
  • Cannot vote twice

✓ Input Validation:
  • All form fields validated
  • Email format checked
  • Required fields enforced
  • Unique constraint on Student ID

✓ Output Encoding:
  • HTML escaping with htmlspecialchars()
  • Prevents XSS attacks
  • All user data escaped before display

================================================================================
TROUBLESHOOTING
================================================================================

Issue: "Connection failed" or "Can't connect to MySQL"
Fix:
  • Start MySQL from XAMPP Control Panel
  • Check username/password in config.php
  • Verify database_setup.sql was executed
  • Check MySQL is listening on localhost:3306

Issue: Pages show blank or 404
Fix:
  • Verify Apache is running
  • Check file paths are correct
  • Look at browser developer console (F12)
  • Check Apache error log

Issue: "Student ID already exists"
Fix:
  • This is normal - Student IDs must be unique
  • Use a different Student ID during registration

Issue: Database tables don't exist
Fix:
  • Open phpMyAdmin: http://localhost/phpmyadmin
  • Go to SQL tab
  • Copy contents of database_setup.sql
  • Execute the SQL

Issue: Styling looks broken
Fix:
  • Verify style.css is in same directory as PHP files
  • Hard refresh browser (Ctrl+Shift+R or Cmd+Shift+R)
  • Check browser console for CSS errors

ADMIN MAINTENANCE
================================================================================

View election statistics:
  SELECT COUNT(*) as total_students FROM users;
  SELECT COUNT(*) as votes_cast FROM votes;
  SELECT (SELECT COUNT(*) FROM votes) / 
         (SELECT COUNT(*) FROM users) * 100 as participation_percent;

Monitor voter turnout:
  SELECT COUNT(*) as voted FROM users WHERE has_voted = 1;
  SELECT COUNT(*) as not_voted FROM users WHERE has_voted = 0;

Check specific candidate votes:
  SELECT president, COUNT(*) as votes FROM votes GROUP BY president;

Reset user (allow re-voting in case of error):
  UPDATE users SET has_voted = 0 WHERE student_id = 'STUDENT_ID';

Export voting results:
  SELECT * FROM votes INTO OUTFILE '/tmp/votes.csv' 
  FIELDS TERMINATED BY ',' LINES TERMINATED BY '\n';
